As i said Earlier, This class got too big and i assigned it with too many responsiblities.
                     The name fooManager is vague, which gives it a meaningless identifier.
                     Probably giving it a more meaningful name could have avoided this.
                     According to what is known as the single responsibility principle,
                     each block should serve exactly one purpose.
                     This is because with a "manager" class you are
                     articficially dividing that responsibility in many respects.
                     Managing many responsibilities instead of one.
